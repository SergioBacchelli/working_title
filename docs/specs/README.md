# docs/specs — индекс спецификаций

Здесь лежат спецификации форматов, контрактов и «словари» проекта.

## Форматы

- `bazis_script/` — спецификация импорта результата расчётов в БАЗИС через Bazis Script:
  - входной JSON-контракт (панели/позиционирование/отверстия),
  - минимальные инварианты и проверки,
  - правила ручного регресса (проверка построения в БАЗИС).

## Как пополняем

- сначала добавляем пример/наблюдение;
- затем превращаем в правило;
- затем фиксируем «минимальный внутренний формат» данных, который должен выдавать модуль;
- затем добавляем валидацию (инварианты, bbox и т.п.).

Если правила не подтверждены реальным примером — помечаем как гипотезу и откладываем до следующего набора тестов.
