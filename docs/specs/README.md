# docs/specs — индекс спецификаций

Здесь лежат спецификации форматов, контрактов и «словари» проекта.

## Форматы

- `bxf2/` — спецификация экспорта BXF2 (панели, координаты, трансформации, далее — отверстия/пазы).
  - `bxf2/BXF2_RULES.md` — текущие правила, зафиксированные по реальному примеру `Novyj-korpus.bxf2`.

## Как пополняем

- сначала добавляем пример/наблюдение;
- затем превращаем в правило;
- затем фиксируем «минимальный внутренний формат» данных, который должен выдавать модуль;
- затем добавляем валидацию (инварианты, bbox и т.п.).

Если правила не подтверждены реальным примером — помечаем как гипотезу и откладываем до следующего файла-образца.
