# Bazis Script — импорт корпуса из JSON

Цель: строить корпус **как набор “деталей” БАЗИС** (панели + отверстия) средствами Bazis Script.

Все вычисления (размеры, позиционирование, сверловка) выполняются на портале.
БАЗИС используется как “исполнитель”: получает готовые данные и создаёт панели/операции в модели.

## Что считаем результатом

- В дереве модели БАЗИС появляются панели (объекты Panel), а не “меши”.
- Панели имеют корректные размеры, ориентацию, имена/ID.
- Отверстия создаются как операции сверления (или эквивалентные объекты API) и привязаны к нужным панелям.

## Минимальный входной контракт (JSON)

Единицы: **мм**.

```json
{
  "meta": { "schema": 1, "unit": "mm" },
  "assembly": { "name": "Cabinet" },
  "parts": [
    {
      "id": "VL",
      "name": "Left",
      "w": 720,
      "h": 560,
      "t": 16,
      "orientation": "vertical",
      "pos": [0, 0, 0],
      "rot": [0, 0, 0]
    }
  ],
  "holes": [
    {
      "partId": "VL",
      "x": 37,
      "y": 100,
      "dia": 5,
      "depth": 12,
      "face": "inner",
      "dir": "-Z",
      "note": "shelf pin"
    }
  ]
}
```

### Пояснения

- `orientation`: логическая ориентация панели в БАЗИС (front / vertical / horizont).
- `pos` / `rot`: размещение панели в пространстве изделия.
- `holes`: координаты **в ЛСК панели** (локальная система), чтобы портал не зависел от “глобальной” ориентации.

> Важно: точная конвенция ЛСК (от какой точки считаем X/Y и как трактуем `face/dir`) фиксируется после первого PoC-импортера и добавляется сюда как правило.

## Регресс-проверка (ручная)

1) В БАЗИС запускаем скрипт импорта.
2) Импортируем эталонный JSON (из `examples/portal_json/`, если папка заведена).
3) Проверяем:
   - количество панелей и имена/ID,
   - размеры (W/H/T),
   - положение в изделии,
   - наличие и параметры отверстий.
