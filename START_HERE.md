# START_HERE.md — инициализация контекста проекта для Рэя

Этот файл — “точка входа”. Он **не** хранит правила проекта и **не** заменяет:
- `README.md`
- `PROJECT_MAP.md`
- `ARCHITECTURE.md`
- `DECISIONS.md`
- `WORKFLOW.md`

Задача `START_HERE.md` — заставить Рэя **сначала прочитать нужные документы**, собрать ограничения и только потом отвечать.

---

## 1) Порядок инициализации (обязательно, сверху вниз)

Рэй, перед ответом по проекту `working_title` прочитай:

1. `README.md`  
   Что за проект и цель.

2. `PROJECT_MAP.md`  
   Карта файлов/папок и где что менять.

3. `ARCHITECTURE.md`  
   Слои, границы, взаимодействия модулей, хранение.

4. `DECISIONS.md`  
   Принятые решения и ограничения (“почему так”).

5. `WORKFLOW.md`  
   Процесс работы: Git, правила изменения, правила взаимодействия.

После этого — только документы по теме задачи:

- **Форматы/контракты/словари:** `docs/specs/README.md`
  - Для импорта в БАЗИС через Bazis Script: `docs/specs/bazis_script/README.md`.

- **Интерфейсы экранов (UI-контракты):** `docs/ui/README.md`
  - Шаблон UI-спеки: `docs/ui/UI_TEMPLATE.md`.
  - Справочник типов полей: `docs/ui/UI_FIELD_FORMAT.md`.
  - Спеки модулей: `docs/ui/modules/`.
