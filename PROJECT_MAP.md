 PROJECT_MAP — карта проекта

## 1. Корень репозитория

- `README.md` — общее описание проекта и цели.
- `PROJECT_MAP.md` — карта файлов и папок (этот документ).
- `ARCHITECTURE.md` — описание архитектуры проекта.
- `DECISIONS.md` — журнал важных решений.
- `WORKFLOW.md` — правила совместной работы Сергея и Рэя.
- `docs/` — дополнительная документация (детальные описания модулей и т.п.).
- `src/` — исходный код проекта.
- `scripts/` — вспомогательные скрипты (генерация, утилиты и т.п.).
- `examples/bxf2/` - примеры генерации итогового файла экспорта bxf2

## 2. Папка `src/`

- `src/server/` — серверная логика (API, обработка запросов, работа с данными).
- `src/client/` — фронтенд (UI, взаимодействие с пользователем), если будет использоваться.

(По мере добавления файлов сюда будем дописывать конкретные файлы и их роли.)

## 3. Папка `docs/`

- `docs/` — общая папка под дополнительную документацию.
  - Возможные подкаталоги:
    - `docs/api/` — описание API.
    - `docs/ui/` — схемы интерфейсов.
    - `docs/storage/` — описание локального и внешнего хранилища.

(Создадим подкаталоги по мере необходимости.)

## 4. Папка `scripts/`

- `scripts/` — вспомогательные скрипты (например, генерация отчётов, утилиты миграций и т.п.).

(Пока пусто — будет описано, когда появятся скрипты.)

## 5. Папка `examples/bxf2/`
Файлы примеров итогового файла экспорта
  - `Original*.bxf2` — исходники от Blum (не очищенные)
  - `Cleared*.bxf2` — вручную очищенные, **100% валидные минимальные эталоны**
  - `Generated*.bxf2` — то, что генерируем по правилам, чтобы проверять “не сломали ли минимум”
